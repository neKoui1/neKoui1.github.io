  • 因为CPU在访问内存时并不是逐个访问字节，而是以字长为单位来访问
  • 这样设计CPU的目的是为了减少CPU访问内存的次数，加大CPU访问内存的吞吐量，比如32位的CPU访问内存的单位是4字节，读取8个字节数据只需要读取2次

内存对齐的规则

  1. 结构体变量的首地址能够被其最宽基本类型成员的大小所整除
  2. 结构体每个成员相对结构体首地址的偏移量(offset)都是成员大小的整数倍，如有需要编译器会在成员之间加上填充字节
  3. 结构体的总大小为结构体最宽基本类型成员大小的整数倍，如有需要编译器会在最末一个成员之后加上填充字节

联合体的共享内存

  • 如果按照union的定义，理论上应该占9个字节，实际打印却是16
  • 依然需要字节对齐使所有被包含的基本数据类型的大小整除，9不能够整除double的8字节，所以填充字节，最终结果为16
  • 共享内存后可以用来判断大端小端

空结构体

  • Go语言中的空结构体大小为0
  • 于是我又去试了试C语言与C++中的空结构体
  • C语言中空结构体的大小依然为0，但C++的空结构体内存为1
  • 因为C++标准规定：任何不同的对象不能拥有相同的内存地址
  • no object shall have the same address in memory as any other variable
  • 如果C++对象大小为0，将会导致
  1. 不能通过指针区分不同的数组对象
  2. sizeof(empty)为0导致非法的除0操作
  • 实际上C++中的struct是向下兼容C语言进行的封装，本质上其实是所有字段都是public的class
  • 因此C++中的空类内存大小也是1
