Chapter 1 总览

Intro

  • 跨主机集群的自动部署、扩展以及运行应用程序容器的平台
  • 容器化应用部署
  • 应用扩展
  • From Google
  • Compared with Docker?

Feature

  1. 自动装箱：自动部署应用容器
  2. 自我修复：容器失败则重启，节点出错则重新部署容器
  3. 水平扩展：对应用容器扩大或剪裁
  4. 服务发现
  5. 负载均衡
  6. 滚动更新
  7. 版本回退
  8. 密钥管理
  9. 存储编排

集群架构组件

Master组件

  • api server： Kubernetes控制面的组件， 该组件公开了 Kubernetes API。 API 服务器是 Kubernetes 控制面的前端
  • scheduler：节点调度，选择node节点来部署应用
  • controller manager：处理集群中常规后台任务，一个资源对应一个控制器
  • etcd：是兼具一致性和高可用性的键值数据库，可以作为保存 Kubernetes 所有集群数据的后台数据库

Node组件

  • kubelet：一个在集群中每个节点上运行的代理。 它保证容器都运行在Pod中
  • kube-proxy：提供网络代理，负载均衡等操作

Core

Pod

  • 最小部署单元
  • 一组容器的集合
  • 共享网络
  • 生命周期短

controller

  • 保证预期pod副本的数量（内置
  • 无/有状态应用部署
  • 可以确保所有node运行同一个pod
  • 一次性任务和定时任务

service

  • 将运行在一组Pod上的应用程序公开为网络服务的抽象方法。
